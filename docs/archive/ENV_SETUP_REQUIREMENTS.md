## Immediate Fix (Required Before Any Production Use)

Run this command once on the production server:

```bash
node scripts/generate-password-hash.cjs <your-chosen-password>
```

This outputs two lines. Add them to `.env`:

```env
OPERATOR_PASSWORD_HASH=<output from script>
JWT_SECRET=<output from script>
```

Then restart:

```bash
pm2 restart jarvis-os
# or
node backend/server.js
```

After this, `POST /auth/login` with your password returns a cookie, and all `/runtime/*` routes become accessible.

## Required Variables (Production — System Breaks Without These)

### `JWT_SECRET`
- **Purpose:** Signs and verifies operator session tokens
- **Generate:** `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
- **Effect if missing:** All `/runtime/*` routes return `503 Auth not configured`. Operator console is completely inaccessible.
- **Requirements:** Any random 32+ byte hex string. Must not change across restarts (doing so invalidates all active sessions).

### `OPERATOR_PASSWORD_HASH`
- **Purpose:** Verifies the operator login password
- **Generate:** `node scripts/generate-password-hash.cjs <password>`
- **Effect if missing:** `POST /auth/login` returns `503 Auth not configured`.
- **Format:** `<16-byte-salt-hex>:<64-byte-scrypt-hash-hex>` — generated by the script above.

### `NODE_ENV`
- **Value:** `production`
- **Effect if missing:** Dev passthrough activates — auth is bypassed, any caller has operator access. Never leave unset on a public-facing server.

### `GROQ_API_KEY`
- **Purpose:** LLM inference for `/jarvis` and `/ai/chat`
- **Effect if missing:** All AI routes disabled. Server starts but returns errors on AI requests.
- **Get it:** console.groq.com

## Nginx Assumptions

The server assumes nginx is handling:
- TLS termination (HTTPS on 443)
- HSTS headers
- CSP headers
- Rate limiting (30r/s per IP)
- Proxy to Node.js on `127.0.0.1:5050`

If nginx is not present:
- Cookies set with `secure: true` will not be sent over plain HTTP
- Login will succeed but the cookie will be silently rejected by the browser
- The operator console will loop at the login screen with no error message

Set `NODE_ENV=development` if running without nginx/TLS to disable the `secure` cookie flag.
