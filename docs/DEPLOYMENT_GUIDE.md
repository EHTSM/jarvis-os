## First Deploy

### 1. Clone and install

```bash
git clone <repo-url> /opt/jarvis-os
cd /opt/jarvis-os
npm ci --omit=dev
npm ci --prefix frontend
```

### 2. Configure environment

```bash
cp .env.example .env
nano .env   # fill in all REQUIRED values
```

Required values (server will fail to start without these):
- `GROQ_API_KEY` — from https://console.groq.com
- `BASE_URL` — your public HTTPS domain, e.g. `https://jarvis.yourdomain.com`
- `PORT` — default `5050`

### 3. Generate operator credentials

```bash
node scripts/generate-password-hash.cjs 'your-secure-password'
```

Copy the output into your `.env`:
```
OPERATOR_PASSWORD_HASH=<output from above>
JWT_SECRET=<output from above>
```

### 4. Set up PM2 startup hook

```bash
pm2 startup   # follow the printed command
```

### 5. Deploy

```bash
bash deploy.sh
```

This will:
- Validate `.env`
- Build the React frontend
- Start JARVIS under PM2 in production mode
- Verify the health endpoint is responding

### 6. Configure nginx

Copy the provided nginx config:

```bash
sudo cp deploy/nginx-jarvis.conf /etc/nginx/sites-available/jarvis
sudo ln -s /etc/nginx/sites-available/jarvis /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

Then set up TLS with certbot:

```bash
sudo certbot --nginx -d jarvis.yourdomain.com
```

## Routine Updates

```bash
git pull
bash deploy.sh          # rebuilds frontend + reloads PM2 (zero-downtime)
# or, backend-only change:
bash deploy.sh --no-build
```

PM2's `reload` command performs a zero-downtime restart — new requests go to the new process before the old one is killed.

## PM2 Reference

```bash
pm2 status              # process list + health
pm2 logs jarvis-os      # tail live logs
pm2 logs jarvis-os --lines 200  # last 200 lines
pm2 restart jarvis-os   # hard restart (brief downtime)
pm2 reload jarvis-os    # zero-downtime reload
pm2 stop jarvis-os      # stop without removing
pm2 delete jarvis-os    # stop and remove from PM2 list
```

Log files are at `logs/out.log` and `logs/err.log`. Rotated at 10 MB, 5 files kept.

| Variable | Required | Description |
|----------|----------|-------------|
| `GROQ_API_KEY` | YES | Groq LLM API key |
| `BASE_URL` | YES | Public HTTPS URL (no trailing slash) |

## Operator Auth

The Runtime panel (`/runtime` routes + frontend Runtime tab) is protected behind a password.

**In development** (no `JWT_SECRET` set): access is open, no login required.

**In production** (with `JWT_SECRET` and `OPERATOR_PASSWORD_HASH`):
- Navigate to the Runtime tab → shows login form
- Enter your operator password → issues an 8-hour httpOnly cookie
- Cookie is renewed on each page load while valid
- Sign out clears the cookie

To rotate the password:
```bash
node scripts/generate-password-hash.cjs 'new-password'
# update OPERATOR_PASSWORD_HASH in .env
bash deploy.sh --no-build
```

## Health Monitoring

The `/health` endpoint is public and returns JSON:

```json
{
  "status": "ok",
  "uptime_seconds": 3600,
  "memory": { "heap_used_mb": 120, "heap_total_mb": 180 },
  "services": { "groq": true, "telegram": true, "whatsapp": false }
}
```

Set up a UptimeRobot (free) monitor at `https://yourdomain.com/health` to get alerts when the server goes down.

## Data Files

Data is stored in `data/`. Back up these files:

| File | Contains | Backup frequency |
|------|----------|-----------------|
| `data/leads.json` | CRM leads | Daily |
| `data/task-queue.json` | Scheduled tasks | On deploy |
| `data/context-history.json` | AI conversation context | Weekly |
| `data/memory-store.json` | Execution history | Weekly |

Quick backup:
```bash
tar -czf backups/data-$(date +%Y%m%d).tar.gz data/
```

## Rollback

See `deploy/rollback.sh` for the automated rollback procedure, or:

```bash
git log --oneline -10          # find the last good commit
git checkout <commit-hash>     # switch to it
bash deploy.sh --no-build      # redeploy backend only
```

To roll back the frontend build only, the previous build is preserved in `backups/` if you ran `deploy/rollback.sh` before upgrading.

## Troubleshooting

**Server won't start:**
```bash
node backend/server.js   # run directly to see full error
```

**PM2 keeps restarting:**
```bash
pm2 logs jarvis-os --err   # look at error log
```

**Runtime tab shows 401:**
- Set `JWT_SECRET` and `OPERATOR_PASSWORD_HASH` in `.env`
- Or leave them unset to bypass auth in non-production mode

**Memory growing above 400 MB:**
- Normal memory range is 120-350 MB
- Check `data/memory-store.json` size — if > 500 KB, run:
  ```bash
  node -e "const f='data/memory-store.json'; const d=JSON.parse(require('fs').readFileSync(f)); require('fs').writeFileSync(f, JSON.stringify(d.slice(-20), null, 2))"
  ```
- Check `data/workflow-checkpoints/` — safe to delete entirely if it exists
