## Mission

Jarvis can take a goal, track it over time, execute related work, and measure completion. No AI in the planning layer — milestones and tasks are generated by deterministic templates refined by keyword analysis.

---

## What Was Built

### `agents/runtime/goalEngine.cjs`

Single file. All goal operations are synchronous except `executeGoalTask` (which fires projectRunner async).

| Entry Point | Purpose |
|---|---|
| `createGoal(opts)` | Create goal with auto-generated milestones + tasks |
| `getGoal(goalId)` | Retrieve by ID |
| `listGoals(opts)` | Filter by type, status, blueprintId, tags |
| `advanceTask(goalId, taskId, result)` | Record task outcome, recompute progress + health |
| `startTask(goalId, taskId)` | Mark task as running |
| `executeGoalTask(goalId, taskId, opts)` | Run task via projectRunner (async) |
| `completeGoal(goalId, opts)` | Mark done, generate completion report |
| `abandonGoal(goalId, reason)` | Mark abandoned |
| `pauseGoal / resumeGoal` | Suspend and restore active state |
| `getHealthScore(goalId)` | 0–100 health score with 4 dimensions |
| `getCompletionReport(goalId)` | Full completion report |
| `getGoalSummary()` | Aggregate counts by type and status |

**Reuses (fail-safe):**
- `unifiedMemoryEngine.index()` — registers goal in cross-reference index (via `setImmediate`)
- `unifiedMemoryEngine.getProjectMemory()` — enriches dev goals with blueprint context
- `projectRunner.runProject()` — executes task descriptions as engineering projects
- `blueprintGenerator.getBlueprint()` — attaches blueprint context to dev goals
- `pipelineOrchestrator.run()` — available for single-task pipelines

**Storage:** `data/goals.json` — max 200, newest-first, atomic write

## Goal Types

| Type | Milestone Sequence |
|---|---|
| `development` | Design → Build → Verify → Deploy → Monitor |
| `business` | Research → Plan → Execute → Measure → Optimize |
| `operational` | Assess → Fix → Verify → Document → Prevent |
| `personal` | Define → Start → Build habit → Review → Complete |

Type is **inferred from title keywords** when not explicitly provided:
- `build`, `deploy`, `api`, `feature`, `code` → `development`
- `revenue`, `customer`, `market`, `growth` → `business`
- `health`, `incident`, `monitor`, `stabilize` → `operational`
- Default → `personal`

---

## Milestone Generation

- Full goal → 5 milestones, 2 tasks each = 10 tasks total
- `quick` / `simple` / `fix` keywords → 3 milestones (trimmed)
- `research` / `audit` focus → 2 milestones (early stages only)
- `monitor` / `observe` operational focus → last 3 milestones

---

## Health Scoring (0–100)

| Dimension | Max | Scoring |
|---|---|---|
| `velocity` | 30 | `(tasks/day) / 2 × 30` — actual vs 2 tasks/day expected |
| `momentum` | 25 | `25 → 0` based on days since last update: <1d=25, <3d=20, <7d=10, ≥7d=0 |
| `focus` | 25 | `(1 - failedRate) × 25` |
| `alignment` | 20 | Full 20 unless milestones done out of sequence |

---

## Progress Tracking

- `completionPct = completedTasks / totalTasks × 100`
- Milestone `status` updates: `pending` → `active` → `completed`
- Recomputed on every `advanceTask` call
- `healthScore` recomputed simultaneously

---

## Completion Report Fields

| Field | Description |
|---|---|
| `totalTasks` | All tasks in goal |
| `completedTasks` | Successfully completed |
| `failedTasks` | Tasks that failed |
| `completionPct` | % done |
| `milestonesCompleted` | Count of fully completed milestones |
| `durationDays` | Days from created to completed |
| `healthScore` | Final health score |
| `summary` | One-line human-readable summary |
| `projectRunIds` | Any projectRunner runs triggered during execution |

## Bug Fixed

`completeGoal()` didn't check for `abandoned` status, allowing completion of abandoned goals. Fixed by adding an explicit guard:
```js
if (goal.status === "abandoned") return { ok: false, error: "cannot complete — goal is abandoned" };
```

## HTTP Routes Added — `backend/routes/ops.js`

| Method | Path | Description |
|---|---|---|
| `POST` | `/goals` | Create a goal (body: title, description, type, targetDate, blueprintId, tags) |
| `GET` | `/goals` | List goals (filter: type, status, blueprintId, limit) |
| `GET` | `/goals/summary` | Aggregate counts by type and status |
| `GET` | `/goals/:id` | Single goal by ID |
| `POST` | `/goals/:id/advance` | Record task outcome (body: taskId, ok, detail, error) |
| `POST` | `/goals/:id/complete` | Complete the goal (body: note) |
| `POST` | `/goals/:id/abandon` | Abandon the goal (body: reason) |
| `GET` | `/goals/:id/report` | Get completion report |
